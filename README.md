## 0. 方案总则

本方案为可直接用于开发、代码落地、产品上线的执行级文档。本版本完成全链路去中心化升级，涵盖：算力调度、任务运算、账号管理、资产消费、收益结算、C2C 转账交易全部去中心化。

**核心定位**：全链路去中心化、本地免费算力可自主开关 + 使用率限制、远程付费算力混合协同、分布式并行运算、远程节点热插拔无感容错、TPT 精准计量、去中心化结算转账、任务完结结算收益。

### `[OPT]` 与原方案的关键差异（修订说明）

| 编号 | 原方案 | 优化点 | 修订理由 |
|------|--------|--------|----------|
| O1 | CUDA 代理劫持，本地无感适配 | **任务 SDK + 沙箱执行**：需求方通过 SDK 提交预定义任务类型（图像处理 / 矩阵运算 / ML 推理 / 哈希校验），贡献端在受控沙箱内执行 | CUDA 劫持需内核驱动 hook，技术上不可靠、安全风险极高，且无法跨任务类型复用；业界（Folding@Home / BOINC / Golem）均采用任务 SDK 模式 |
| O2 | 服务端仅做加密存证，无任何逻辑 | **Bootstrap 节点 + Witness 见证网络**：保留只读存证服务作为 DHT 引导节点；引入轮换见证节点网络对结算/转账做去中心化公证 | 纯无引导节点的 P2P 无法完成首次连接；纯本地记账无防双花机制 |
| O3 | 邮箱注册 + 验证码 30 分钟有效，去中心化闭环 | **邮箱 + Ed25519 密钥对 + 助记词**：邮箱为可读标识，密钥对为权属根；助记词 (BIP-39) 用于本地备份恢复；3-of-5 社交恢复作为兜底 | 无任何中心服务时丢密钥即丢号；纯邮箱验证码仍需邮件服务器 |
| O4 | 1TPT = 1TOPS·1Min，按额定 TOPS 计费 | **实测吞吐量计费**：TPT = 实测吞吐量 (TOPS) × 显存占用率 × 有效时长 (分钟)；任务前用基准测试校准该节点真实 TOPS | 额定 TOPS 是厂商理论值，与实际任务吞吐差异巨大，易产生计费争议 |
| O5 | 贡献者完整完成任务自动结算，异常清零 | **冗余执行 + 挑战期结算**：每个子任务至少 2 节点冗余执行，结果哈希一致才结算；结算进入 1 小时挑战期，期间见证节点可发起重算验证 | 单节点执行无法证明"真算了"，存在空跑骗补风险 |
| O6 | 转账 2% 手续费，1% 运维 + 1% 收益 | **2% 手续费分配**：1% 见证网络激励池（去中心化分润给见证节点）+ 0.5% 运维池 + 0.5% 收益池 | 见证节点需激励否则无人公证；分润更均衡 |
| O7 | 平台无账号管控、无封禁、无冻结权限 | **保留紧急熔断权（仅限违法内容）**：去中心化任务风控 + 内容哈希黑名单；冻结需 5 个见证节点多重签名 | 完全无管控在合规层面无法落地（违法任务必须可阻断） |
| O8 | TPT 不可提现、不可上链 | 维持不变，但增加 **任务保证金 + Slashing**：贡献者需质押少量 TPT 才能接任务，作恶（虚假结果/中途恶意退出）扣除质押 | 无 Slashing 无法防止 Sybil 攻击与恶意节点 |

## 1. 产品核心定位与核心优势

### 1.1 产品定位

基于 P2P 去中心化组网的 Windows 端 GPU 算力共享平台，搭建完全去中心化的全民闲置 GPU 算力供需与资产流通市场。算力调度、任务处理、账号校验、资产消费、收益清算、用户转账全部由客户端点对点去中心化执行，服务端仅做兜底存证与 DHT 引导。

### 1.2 核心差异化优势

- **全链路去中心化架构**：算力、账号、资产、结算、转账、交易全部去中心化，公开透明、不可篡改、自主可控
- **精细化算力可控机制**：需求端可自主开关本地算力、限制本地 GPU 使用率；供给端可自定义共享 GPU 使用率
- **精准 TPT 计量体系**：基于实测吞吐量的 TPT 计量，公式透明、计费精准
- **冗余验证 + 挑战期结算**：双节点冗余执行 + 见证网络挑战期，杜绝虚假算力
- **远程无感容错机制**：任意远程节点掉线、退出、崩溃，不中断本地主任务
- **`[OPT]` Slashing 防作恶**：贡献者质押机制 + 恶意行为扣除质押，经济模型自洽
- **分布式大算力能力**：低配本地设备可运行超大规模算力任务

## 2. 全局固化规则（开发强制标准）

### 2.1 TPT 算力计量体系（`[OPT]` 实测校准版）

- **通用算力计量单位**：TPT（唯一平台资产）
- **单位定值标准**：1 TPT = 1 TOPS · 1 Min
- **`[OPT]` 消费精准计算公式**：
  ```
  用户消费 TPT = 实测吞吐量(TOPS) × 显存占用率 × 有效运算时长(分钟)
  实测吞吐量 = 任务前 30 秒基准测试得出（任务类型相关）
  ```
- **计费对象**：仅远程共享算力计量扣费，用户本地 GPU 算力默认免费、不计 TPT
- **结算分润**：需求方远程算力消费总额扣除 2% 手续费，98% 对应 TPT 收益点对点结算给合规完成任务的算力贡献者
- **`[OPT]` 手续费拆分**：2% = 1% 见证网络激励池 + 0.5% 运维池 + 0.5% 收益池

### 2.2 TPT 资产去中心化规则

- **平台唯一资产**：TPT，全程客户端去中心化记账、确权、流转
- **流通范围**：平台内远程算力消费、点对点转账、C2C 交易、算力收益结算
- **合规属性**：不可提现、不可上链、不可炒作、无金融属性，纯内部算力计量与流通凭证
- **充值规则**：法币充值 0 手续费，全额到账，充值后资产由客户端本地确权，服务端仅存证订单记录
- **资产归属**：用户 TPT 资产所有权、使用权、转账权完全归用户

### 2.3 双端 GPU 使用率控制规则

#### 2.3.1 算力需求端规则

- 自主启停本地算力：需求者可手动选择「开启本地 GPU 协同运算 / 关闭本地 GPU 运算」
- 本地算力使用率限制：开启本地算力时，支持自定义 GPU 最大使用率阈值（0%-100% 可调）
- 限制优先级：用户自定义使用率优先级最高，客户端本地调度不得超出用户设定阈值

#### 2.3.2 算力贡献端规则

- 共享算力使用率限制：贡献者开启算力共享后，可自定义对外共享的 GPU 最大使用率阈值
- 系统强制约束：客户端本地远程调度、任务分配时，严格不超出贡献者设定的 GPU 使用率上限
- **`[OPT]` 质押门槛**：贡献者需质押 ≥ 100 TPT 才能接远程任务，作恶扣质押

### 2.4 去中心化收益结算硬性规则（`[OPT]` 冗余验证版）

- **收益获取条件**：
  1. 算力贡献者完整、持续完成单次算力任务
  2. **`[OPT]` 冗余执行结果哈希与对端节点一致**（或通过挑战期重算验证）
  3. 进入 1 小时挑战期未被见证节点否决
- **无收益场景**：掉线、主动退出、程序崩溃、死机、断网等异常；**`[OPT]` 结果哈希不一致且重算证实作弊**
- **计量终止规则**：异常瞬间停止 TPT 计量，无虚假计费、无部分结算
- **任务接续规则**：贡献者中途异常，客户端自动触发容错补位，剩余任务由其他节点/本地算力接续
- **`[OPT]` Slashing 规则**：恶意伪造结果，扣除本次质押的 50%；连续 3 次作弊，扣除全部质押并加入黑名单
- **结算方式**：去中心化点对点结算 + 见证网络公证，无平台中转、无人工审核、挑战期满实时到账

### 2.5 任务运行基础规则

- 算力优先级：默认本地算力优先（用户可手动关闭），负载溢出自动扩容远程算力
- 任务形态：纯远程运算、本地+远程混合运算、多节点分布式并行运算
- 故障规则：远程子任务故障隔离，本地主任务永久不中断
- 计量终止：节点异常瞬间停止 TPT 计量，仅结算完整有效任务
- **`[OPT]` 任务类型**：图像处理、矩阵乘法、ML 推理、哈希校验（任务 SDK 内置类型）

## 3. 全模块去中心化核心机制

### 3.1 去中心化混合算力调度机制

客户端实时监测本地显存、TOPS 负载、运行耗时，结合用户配置，本地闭环完成四级去中心化调度：

- **本地算力关闭模式**：点对点调度远程算力，纯付费执行
- **轻量任务**：按用户设定使用率调用本地算力，零成本运行
- **中量任务**：本地负载触达阈值，自动点对点拉起空闲远程节点，形成混合算力池
- **重量任务**：本地满载后，客户端去中心化拆分任务、分布式分发、本地聚合结果

### 3.2 去中心化节点容错机制

- 任务完全解耦：本地主任务与远程子任务相互隔离，互不影响
- 客户端本地毫秒级心跳监测所有 P2P 节点状态
- 异常节点自动剥离、终止子任务、停止计量
- 自动补位、负载迁移，全程去中心化自愈，无服务端干预
- **`[OPT]` 断点续算**：子任务进度每 10 秒 checkpoint 一次，新节点接续从最近 checkpoint 继续

### 3.3 去中心化 TPT 计量与消费机制

- 本地实时采样硬件参数，严格按公式计量 TPT 消耗
- 任务前本地冻结对应 TPT 额度 + **`[OPT]` 1.1 倍冗余冻结**（防任务超时超额）
- 任务完成自动点对点扣费、手续费拆分、额度释放
- 异常场景自动校准、撤销未生效计量，保障消费公平
- 消费记录本地确权，同步加密存证至见证网络，不可篡改

### 3.4 去中心化收益结算机制

- 任务完整结束后，客户端自动核算本次任务总 TPT 消耗
- **`[OPT]` 冗余节点结果哈希比对**，一致才进入结算
- 本地自动拆分 2% 手续费、98% 贡献收益
- 点对点直接划转收益至贡献者账户
- **`[OPT]` 1 小时挑战期**，期间见证节点可触发重算验证
- 结算日志本地确权 + 见证网络加密存证，可追溯、不可篡改

### 3.5 去中心化账号管理机制（`[OPT]` 密钥对版）

- 邮箱为可读身份标识，**Ed25519 密钥对为权属根**
- **`[OPT]` BIP-39 助记词**：注册时生成 12 词助记词，用户离线保存，可恢复全部资产
- **`[OPT]` 3-of-5 社交恢复**：用户指定 5 个信任联系人，丢失密钥时由 3 人联合签名恢复
- 密码加密本地存储 + 云端加密备份
- 平台无账号管控、无封禁权限（**`[OPT]` 仅违法内容可由 5 见证节点多签冻结**）
- 用户可自主本地注销账号、清空本地数据

### 3.6 去中心化 C2C 转账与交易机制

- 用户自由点对点转账 TPT，客户端本地校验余额、执行划转
- 转账 2% 手续费自动扣除，内部分润固化
- C2C 挂单、撮合、成交全部客户端去中心化完成
- **`[OPT]` 撮合账本**：订单簿广播至 P2P 网络，最新价格由最长链决定
- 资产流转实时到账、本地确权、见证网络加密存证
- 平台无权冻结、拦截、逆转任何用户转账与交易行为

## 4. 双端去中心化完整流程

### 4.1 算力贡献者（供给端）

1. 邮箱注册登录，生成 Ed25519 密钥对 + 助记词
2. **`[OPT]` 质押 ≥ 100 TPT**（新用户赠送 200 TPT 福利额度）
3. 客户端本地扫描 GPU 参数，自定义共享 GPU 使用率阈值
4. 开启算力共享，节点去中心化入网 P2P 算力池
5. 被动接收需求端点对点下发算力任务
6. **`[OPT]` 与另一冗余节点并行执行**，提交结果哈希
7. 完整完成任务 + 哈希一致，客户端自动点对点结算 TPT 收益
8. 1 小时挑战期满，收益最终到账
9. 收益到账后可自主转账、交易、留存

### 4.2 算力需求者（使用端）

1. 邮箱登录，自主管理本地 TPT 资产，余额不足可法币充值
2. 自主配置本地算力开关、GPU 使用率上限
3. 发起算力任务（通过任务 SDK 选择任务类型 + 上传输入数据）
4. 本地前置冻结对应 TPT 额度（含 1.1 倍冗余）
5. 任务运行中自动容错、自动补位、无感稳定运行
6. 任务完成自动计量、自动扣费、自动结算收益给贡献节点
7. 剩余额度自动解冻，账单本地确权 + 见证网络存证

## 7. 去中心化安全体系

### 7.1 贡献端设备安全

- 自定义显存预留、GPU 使用率上限，设备自主防护
- 高温、过载自动熔断暂停共享

- 运算数据端对端加密，远端不留存、不缓存
- 去中心化资产冻结机制，杜绝超额扣费、虚假扣费
- 客户端自动屏蔽异常节点，保障任务稳定
- 所有资产流转、消费、结算均为用户自主行为
- 异常任务自动终止计量、自动作废结算

## 9. 最终技术架构

- **组网**：DHT 去中心化 P2P 分布式组网（Kademlia），Bootstrap 节点引导
- **传输**：NaCl Box 端对端加密直连，无中间服务器转发
- **算力层**：**`[OPT]` 任务 SDK + 沙箱执行**（替代 CUDA 劫持）
- **核心逻辑**：客户端原生实现算力调度、任务拆分、容错自愈、TPT 计量、去中心化结算、点对点转账、账号校验
- **`[OPT]` 共识层**：见证网络 PBFT-lite 公证，最长链 + 多签确认
- **服务端唯一职责**：DHT Bootstrap、加密数据存证、订单记录、脱敏备份
- **运行环境**：Windows 10/11 64 位
- **打包**：标准 EXE 安装包（PyInstaller 打包）

### `[OPT]` 9.1 实现技术栈

| 模块 | 技术 |
|------|------|
| P2P 组网 | Python asyncio + 自实现 Kademlia DHT |
| 加密 | PyNaCl（Ed25519 + X25519 + SecretBox） |

| 任务执行 | 子进程沙箱 + NumPy/CuPy 回退 |

| 前端 | 原生 HTML/CSS/JS + Chart.js |
| 打包 | PyInstaller → EXE |
